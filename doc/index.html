redsys-256

This package allows to create a Redsys payment form with the new (December 2015) encryption protocol.

Description

The containing "redsys" package defines two Python classes:

  • Transaction
  • Commerce

The first one represents a payment transaction. The second one represents the commerce with its secret key and FUC.

Test

In order to get the encoded parameters and the signature, you must only:

from redsys import Commerce, Transaction c = Commerce(secret_key=YOUR_KEY, fuc=YOUR_MERCHANT_CODE, terminal=TERMINAL, url=A_NOTIFICATION_URL) t = Transaction(amount=100, order='301215093032', url_ok='http:/example/ok', url_ko='http://example/ko') c.generate_form(t)

This returns a dict with:

  • Ds_SignatureVersion
  • Ds_Signature
  • Ds_MerchantParameters
